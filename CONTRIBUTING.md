# How to contribute to HALLucy?

> [!NOTE]
>
> Please note that this project is a personal project. If someone wants to contribute, please take your time to understand the codebase prior to submitting any pull requests. AI is a great tool, but it must be used responsibly, and transparency in using it is important.

This document describes the process of contributing to the HALLucy project.

Generally speaking, **anyone** can contribute to the project, may that be opening an **issue**, **discussion** or **pull request**. 

My aim is to maintain a qualitative code base, and therefore I won't accept drive-by pull requests. 
A drive-by pull request is any pull request with low effort and low engagement by the submitter. 
Ai based PRs and agent-generated slop is a type of drive-by PR.

To allow for outside contributions and accomplish my goal, some requirements apply to outside contributions.

## Requirements for outside contributions

> [!IMPORTANT]  
> 
> 1. **You *must* understand your code.** AI assistance is welcome, but the resulting code *must* be fully reviewed, understood and edited by a human.
> 2. **Any AI usage *must* be disclosed.** The used tools (e.g. Copilot, Claude, etc) along with the extent of the AI's contribution, *must* be disclosed in the PR.
> 3. **PR *must* be linked to an approved issue.** Only pull requests associated with a previously accepted issue may be approved. Always reference an existing issue in your PR.

These requirements apply only to outside contributions to HALLucy. 
The author and future maintainers are exempt from these requirements as they've proven themselves trustworthy to apply good judgement.
